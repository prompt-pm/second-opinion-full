# Adaptive Decision Flow Specification

## Overview

An intelligent, adaptive user flow for decision-making that deploys the right tool at the right time based on where users get stuck. The system helps users through three phases: **Identify** what matters, **Rank** priorities, and **Commit** to trade-offs.

### Design Principle

> The magic isn't the tools themselvesâ€”it's knowing when to deploy them.

The flow uses **progressive disclosure based on struggle signals**. Confident users get a fast path (~2 min), while stuck users get deeper scaffolding (~5-10 min).

---

## User Archetypes

| Archetype | Behavior | Needs |
|-----------|----------|-------|
| **Confident Decider** | Knows what they want, just needs structure | Fast path, validation |
| **Values Uncertain** | Can't articulate what matters | Story prompts, card sort |
| **Ranking Paralyzed** | Knows values but "everything's important" | Pairwise comparisons, elimination |
| **Commitment Anxious** | Has ranked but can't pull the trigger | Trade-off acknowledgment, regret inoculation |

## Struggle Detection System

### Metrics to Track

```typescript
interface UserBehaviorMetrics {
  // Time-based
  timeOnCurrentStep: number;
  totalSessionTime: number;

  // Interaction-based
  clickCount: number;
  reorderCount: number;  // For ranking
  backNavigationCount: number;

  // Content-based
  inputLength: number;
  itemsListed: number;

  // Explicit signals
  clickedHelpButton: boolean;
  clickedUnsureButton: boolean;
  clickedEquallyImportant: boolean;
  clickedNeedMoreTime: boolean;
}
```

### Detection Rules

```typescript
const STRUGGLE_THRESHOLDS = {
  // Phase 1: Identification
  identify: {
    tooFewItems: { itemsListed: '<2', timeMs: '>60000' },
    tooManyItems: { itemsListed: '>5' },
    explicitUnsure: { clickedUnsureButton: true },
  },

  // Phase 2: Ranking
  rank: {
    cantDifferentiate: { clickedEquallyImportant: true },
    oscillating: { reorderCount: '>3', timeMs: '<30000' },
    paralysis: { timeMs: '>45000', clickCount: '<2' },
  },

  // Phase 3: Commitment
  commit: {
    hesitation: { timeMs: '>30000', clickCount: '<1' },
    secondGuessing: { backNavigationCount: '>0' },
    explicitUncertain: { clickedNeedMoreTime: true },
  },
};
```

## API Endpoints (New)

### Session Management

```
POST /api/decision/start
  Body: { decision: string, stuckLevel: string }
  Response: { sessionId: string, nextStep: string }

GET /api/decision/:sessionId
  Response: { session: DecisionSession, state: CurrentState }
```

### Phase 1: Identification

```
POST /api/decision/:sessionId/identify/quick
  Body: { priorities: string[] }
  Response: { priorities: Priority[], nextStep: string }

POST /api/decision/:sessionId/identify/stories
  Body: { bestCase: string, worstCase: string }
  Response: { extractedPriorities: Priority[], nextStep: string }

GET /api/decision/:sessionId/identify/cards
  Response: { cards: Card[], decisionType: string }

POST /api/decision/:sessionId/identify/cards
  Body: { selectedCards: string[], customPriorities: string[] }
  Response: { priorities: Priority[], nextStep: string }
```

### Phase 2: Ranking

```
POST /api/decision/:sessionId/rank/quick
  Body: { rankedPriorityIds: string[] }
  Response: { ranking: RankedPriority[], nextStep: string }

GET /api/decision/:sessionId/rank/tournament
  Response: { matchups: Matchup[], currentMatchup: number }

POST /api/decision/:sessionId/rank/tournament/matchup
  Body: { matchupId: string, winnerId: string }
  Response: { nextMatchup?: Matchup, complete: boolean, ranking?: RankedPriority[] }

POST /api/decision/:sessionId/rank/budget
  Body: { weights: { priorityId: string, weight: number }[] }
  Response: { finalRanking: RankedPriority[], nextStep: string }

POST /api/decision/:sessionId/rank/eliminate
  Body: { eliminatedId: string }
  Response: { remaining: Priority[], complete: boolean, ranking?: RankedPriority[] }
```

### Phase 3: Commitment

```
GET /api/decision/:sessionId/recommendation

POST /api/decision/:sessionId/commit/tradeoff
  Body: { accepted: boolean }

POST /api/decision/:sessionId/commit/premortem
  Body: { canLiveWithRisk: boolean, escapePlan?: string[] }
  Response: { nextStep: string }

POST /api/decision/:sessionId/commit/finalize
  Body: { finalChoice: string }
```

## Frontend State Machine

```typescript
type FlowState =
  | { phase: 'entry' }
  | { phase: 'identify', tool: '1.0' | '1A' | '1B' }
  | { phase: 'rank', tool: '2.0' | '2A' | '2B' | '2C' }
  | { phase: 'commit', tool: '3.0' | '3A' | '3B' | '3C' }
  | { phase: 'done' };

interface FlowContext {
  sessionId: string;
  decision: string;
  stuckLevel: string;
  priorities: Priority[];
  ranking: RankedPriority[];
  recommendation?: Recommendation;
  tradeOffs?: TradeOffs;
  receipt?: DecisionReceipt;
  behaviorMetrics: UserBehaviorMetrics;
}

// Transitions
type FlowEvent =
  | { type: 'START', decision: string, stuckLevel: string }
  | { type: 'IDENTIFY_COMPLETE', priorities: Priority[] }
  | { type: 'IDENTIFY_STRUGGLE', signal: string }
  | { type: 'RANK_COMPLETE', ranking: RankedPriority[] }
  | { type: 'RANK_STRUGGLE', signal: string }
  | { type: 'COMMIT_ACCEPTED' }
  | { type: 'COMMIT_STRUGGLE', signal: string }
  | { type: 'FINALIZE', choice: string };
```

## Success Metrics

### Primary Metrics
- **Completion Rate**: % of sessions that reach a decision
- **Time to Decision**: Average time from start to commitment
- **Tool Deployment Accuracy**: % of times deeper tools helped vs. added friction

### Secondary Metrics
- **Struggle Detection Accuracy**: True positive rate of detecting stuck users
- **Tool Effectiveness**: Completion rate after tool deployment
- **User Satisfaction**: Post-decision confidence rating
